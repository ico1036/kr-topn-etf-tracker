# 데이터 신뢰도 루브릭

압축 ETF 트래커가 수집하는 모든 데이터 조각(예: "TIGER 반도체TOP10의 다음 리밸일 = 2026-10-16")은 아래 4개 축으로 채점하고 최종적으로 **High / Med / Low** 하나로 접는다. 목적은 필드 단위로 출처·confidence를 강제하는 것.

## 축 1 — 출처 권위 (Source Tier)

| Tier | 정의 | 예 |
|---|---|---|
| **T0** 1차/공식 | 발행 주체가 법적 책임지고 내는 원본 | KRX 일일 PDF(자산구성내역), 운용사 공식, 지수 방법론 Book |
| **T1** 공식 파생 | 공식 데이터를 재배포하는 신뢰 채널 | 운용사 상품페이지, FnIndex, 네이버금융, funetf |
| **T2** 2차 편집 | 사람이 가공한 리포트/기사 | 증권사 리포트, Smartkarma, 언론 |
| **T3** 비공식/추정 | 커뮤니티·블로그·LLM 추론 | 카페글, 요약 스니펫, 추정 |

**원칙: 정량 필드(비중·수량·AUM·리밸일)는 T0/T1만 값으로 채택.** T2/T3는 교차검증 보조로만.

## 축 2 — 추출 방식 신뢰도 (Extraction Reliability)

| 등급 | 정의 | 리스크 |
|---|---|---|
| **E0** 구조화 원본 | CSV/JSON/API 필드 직접 파싱 | 거의 없음 |
| **E1** 표 파싱 | HTML 테이블/PDF 표 규칙 추출 | 컬럼 밀림, 단위 오인 |
| **E2** 본문 텍스트 추출 | 문장에서 값 추출 | 요약 모델 환각·누락 |
| **E3** 규칙 기반 계산 | 방법론+달력으로 계산 | 규칙 오해석 |

**원칙: E2 값은 원문 snippet 동봉, E3 값은 "추정" 태그 필수.**

## 축 3 — 도메인 정합성 검사 (Invariants)

하나라도 실패하면 confidence 강등 + 플래그. 통과/실패를 DB에 함께 적재한다.

**구성종목·비중**
- 비중 합 = 100% ± 0.5% (현금·기타 포함)
- 종목 수 = 지수 규칙 개수 (TOP10 → 10)
- 각 비중 ∈ (0, 100], 상한룰 위반 없음 (예: 반도체TOP10 상위 2종목 각 ≈25% cap)
- 종목코드 유효 + KRX 상장 매칭
- 스냅샷 기준일이 거래일, 조회일과 정합

**리밸런싱 일정**
- 다음 리밸일 > 오늘
- 방법론상 주기와 계산 날짜 일치
- 선물옵션 만기일 규칙(D+1~D+5) 결과가 거래일에 착지
- 직전 정기변경 실제 발생일로 역검증

**AUM·좌수**
- 양수, 전일 대비 급변(±30%↑) 시 플래그

## 축 4 — 교차검증 × 신선도

- **교차검증**: 독립 소스 2곳 이상 일치 → 신뢰 가산. 정량 필드는 2소스 일치를 High 조건으로 요구.
- **신선도(필드별 SLA)**:
  - 구성종목·비중·AUM → 거래일 T+1 이내
  - 리밸 일정 → 정기변경 직후 및 분기 1회, 예정일 D-30 재검증
  - 방법론 → 개정 공표일 추적
- 초과 시 값 유지 + `stale` 플래그 + confidence 한 단계 강등

## 최종 등급 규칙

| 라벨 | 조건 | 엔진 취급 |
|---|---|---|
| 🟢 **High** | T0/T1 + E0/E1 + invariant 전부 통과 + (정량) 2소스 일치 + 신선 | 그대로 사용 |
| 🟡 **Med** | 단일 T0/T1, invariant 통과, 교차검증 미완 or 약간 stale | 표시 + "미교차검증" 배지 |
| 🔴 **Low** | T2/T3만 / invariant 실패 / E2·E3 단독 / stale 초과 | 노출 금지 or "추정" 워터마크, 재수집 큐 |

## Provenance 스키마

각 레코드에 첨부:

```
value, unit, as_of_date,
source_url, source_tier(T0~T3), fetched_at,
extraction(E0~E3), raw_snippet,
invariants_passed[], invariants_failed[],
corroborating_sources[], confidence(High/Med/Low), flags[]
```

## 철학 3줄

- **값과 검증은 별개 상태다** — 있다고 믿는 게 아니라 invariant를 통과해야 믿는다.
- **정량은 T0/T1 + 2소스, 정성은 출처 표기로 충분** — 문턱을 데이터 성격별로 다르게.
- **Low는 버리는 게 아니라 격리** — 재수집 큐로 추적, 웹엔 안 띄움.
